BoatSpotMedia v50.5AO - Buyer Password Reset Fix

Scope: Safe backend-only patch for buyer forgot-password reset flow.

Fixed:
- Buyer reset-password page was showing "password changed successfully" but the old password still worked.
- Cause: reset flow updated password_hash, then attempted to update an optional legacy password column. If the legacy column did not exist, the transaction rollback removed the password_hash update.
- New behavior: reset flow updates only the real login column used by buyers: user.password_hash.
- Token is marked as used only after the password_hash update succeeds.

Not touched:
- Creator settings password flow.
- Owner login.
- Creator login.
- Stripe, R2, uploads, ads, menus, mobile UI, database structure.

Test after deploy:
1. Buyer requests forgot password.
2. Buyer receives reset link.
3. Buyer sets new password.
4. Login with new password should work.
5. Login with old password should fail.
